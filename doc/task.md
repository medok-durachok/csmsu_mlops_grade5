# Описание задачи

## Постановка задачи

Разработка MLOps системы для предсказания страховых выплат по автомобильным полисам с автоматическим переобучением и мониторингом качества. Данные берутся из файла `motor_data14-2018.csv`, содержащего информацию о страховых полисах автомобилей.

### Структура данных

Входные признаки:
- SEX - пол страхователя
- INSR_BEGIN, INSR_END - даты начала и окончания страхования
- EFFECTIVE_YR - год действия полиса
- INSR_TYPE - тип страхования
- INSURED_VALUE - страховая стоимость
- PREMIUM - страховая премия
- OBJECT_ID - идентификатор объекта
- PROD_YEAR - год производства автомобиля
- SEATS_NUM - количество мест
- CARRYING_CAPACITY - грузоподъемность
- TYPE_VEHICLE - тип транспортного средства
- CCM_TON - объем двигателя в тоннаж
- MAKE - марка автомобиля
- USAGE - назначение использования

Целевая переменная:
- CLAIM_PAID - сумма выплаченного страхового возмещения

### Режим обработки данных

Данные обрабатываются инкрементально батчами. Система отслеживает уже обработанные данные через маркерные файлы `.processed`, что позволяет избежать повторной обработки и обеспечивает возможность дообучения модели на новых данных.

## Data Quality и валидация

### Проверка качества данных

Реализован модуль `dq.py` для базовой оценки качества:
- Подсчет количества строк
- Расчет доли дубликатов
- Расчет доли пропущенных значений по каждому признаку
- Выявление константных признаков
- Проверка типов данных

### Очистка данных

Параметры очистки:
- Максимальная доля пропусков: 60%
- Максимальная доля дубликатов: 20%
- Удаление строк с полностью пустыми значениями

### Валидация через ассоциативные правила

Реализован алгоритм Apriori для поиска закономерностей в данных:
- Минимальная поддержка: 0.1
- Минимальная уверенность: 0.5
- Бинаризация признаков для категориальных (топ-5 значений) и числовых (квантили)
- Проверка нарушений найденных правил в новых данных

## Сохранение артефактов

### Структура хранения

```
data/
  raw/              - сырые батчи с метаданными (CSV + JSON)
  processed/        - обработанные данные
  incoming/         - входные данные для обучения и inference

models/
  models_metadata.json           - метаданные всех моделей
  <model_type>_<timestamp>/      - директория конкретной версии
    model.pkl                    - сериализованная модель
    preprocessor.pkl             - препроцессор

outputs/          - результаты предсказаний
reports/          - отчеты мониторинга
```

### Версионирование моделей

Каждая версия модели сохраняется с:
- Уникальным идентификатором: тип_модели + timestamp
- Метриками
- Параметрами обучения
- Временем обучения
- Информацией о данных (количество образцов)

### Метаданные батчей

Для каждого обработанного батча сохраняется JSON с информацией:
- batch_id
- timestamp
- source_file
- chunk_size
- start_row, end_row
- n_rows_processed

## Проектирование модели

### Обработка признакового пространства

#### 1. Идентификация типов признаков
- Автоматическое разделение на числовые и категориальные
- Исключение целевой переменной из обработки

#### 2. Обработка выбросов
Метод IQR для числовых признаков:
- Q1 = 25-й перцентиль
- Q3 = 75-й перцентиль
- IQR = Q3 - Q1
- Границы: [Q1 - 1.5*IQR, Q3 + 1.5*IQR]
- Выбросы обрезаются

#### 3. Обработка пропусков
- Числовые признаки: заполнение медианой
- Категориальные признаки: заполнение модой

#### 4. Кодирование категориальных признаков
- Label Encoding для всех категориальных признаков
- Пропущенные значения заменяются на 'MISSING' перед кодированием

#### 5. Масштабирование
- StandardScaler для числовых признаков

### Используемые модели

#### 1. Линейная регрессия
Тип: LinearRegression

Гиперпараметры:
- fit_intercept: True

#### 2. Дерево решений
Тип: DecisionTreeRegressor

Гиперпараметры:
- max_depth: 10
- min_samples_split: 2
- min_samples_leaf: 1
- random_state: 42

#### 3. Нейронная сеть
Тип: MLPRegressor

Гиперпараметры:
- hidden_layer_sizes: (100, 50)
- max_iter: 500
- alpha: 0.0001
- solver: 'adam'
- early_stopping: True
- validation_fraction: 0.1
- random_state: 42

### Оценка качества моделей

#### Метрики
- MSE
- RMSE
- MAE
- R2 Score

#### Кросс-валидация
- Количество фолдов: 5
- Метрики CV: среднее и стандартное отклонение RMSE и R2

#### Выбор лучшей модели
- Критерий: максимальный R2 Score
- Автоматический выбор из трех обученных моделей
- Сохранение всех моделей для анализа

### Разделение данных
- Тренировочная выборка: 80%
- Тестовая выборка: 20%
- random_state: 42

### Стратегия обучения

#### Начальное обучение
- Обучение на первом батче данных
- Одновременное обучение всех трех моделей
- Выбор и сохранение лучшей модели

#### Инкрементальное обновление
- При поступлении новых батчей данных
- Переобучение всех моделей на новых данных
- Сравнение с предыдущей версией
- Сохранение новой версии при улучшении метрик

### Контроль качества

Реализован модуль QualityControl для:
- Проверки деградации модели
- Валидации новых моделей перед сохранением
- Ведения истории всех версий моделей
- Возможности отката к предыдущим версиям
