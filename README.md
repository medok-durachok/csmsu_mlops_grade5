# MLOps система для предсказания страховых выплат

## Описание

Система машинного обучения для предсказания страховых выплат по автомобильным полисам с автоматическим переобучением и мониторингом качества моделей.

## Реализованные компоненты

### 1. Сбор и обработка данных
- Последовательная загрузка данных из основного файла батчами
- Автоматическое отслеживание обработанных данных
- Сохранение метаданных для каждого батча
- Проверка качества данных (пропуски, дублирования, константные признаки)

### 2. Препроцессинг
- Очистка данных с настраиваемыми порогами для пропусков и дубликатов
- Автоматическое определение типов признаков
- Кодирование категориальных признаков
- Масштабирование числовых признаков
- Обработка пропущенных значений

### 3. Обучение моделей
- Три типа моделей: линейная регрессия, дерево решений, нейронная сеть
- Кросс-валидация для оценки качества
- Автоматический выбор лучшей модели по метрикам
- Сохранение моделей с версионированием

### 4. Хранение и версионирование
- Централизованное хранилище моделей
- Метаданные для каждой версии (метрики, параметры, время обучения)
- Возможность загрузки любой версии модели
- Контроль качества при сохранении новых версий

### 5. Мониторинг и обновление
- Автоматическое переобучение при поступлении новых данных
- Отслеживание деградации качества модели
- История всех версий моделей с метриками

### 6. Ассоциативные правила
- Поиск закономерностей в данных с помощью алгоритма Apriori
- Применение правил для улучшения качества данных

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

### Обучение начальной модели или обновление существующей

```bash
python run.py -mode update
```

Опционально можно указать размер батча:

```bash
python run.py -mode update -chunk_size 5000
```

### Просмотр информации о моделях

```bash
python run.py -mode summary
```

### Предсказания (inference)

```bash
python run.py -mode inference -input_file data/incoming/test_data.csv
```

Опционально можно указать файл для сохранения результатов:

```bash
python run.py -mode inference -input_file data/incoming/test_data.csv -output_file outputs/my_predictions.csv
```

## Конфигурация

Основные параметры в [`config.py`](config.py:1):

- `BATCH_SIZE`: размер батча для обработки данных (по умолчанию 10000)
- `MAX_MISSING_RATE`: максимальный процент пропусков (0.6)
- `MAX_DUPLICATE_RATE`: максимальный процент дубликатов (0.2)
- `TEST_SIZE`: размер тестовой выборки (0.2)
- `CV_FOLDS`: количество фолдов для кросс-валидации (5)

Параметры моделей также настраиваются в конфигурации:

- Линейная регрессия: `LR_FIT_INTERCEPT`
- Дерево решений: `DT_MAX_DEPTH`, `DT_MIN_SAMPLES_SPLIT`, `DT_MIN_SAMPLES_LEAF`
- Нейронная сеть: `MLP_HIDDEN_LAYER_SIZES`, `MLP_MAX_ITER`, `MLP_ALPHA`, `MLP_SOLVER`

## Метрики качества

Система отслеживает следующие метрики:

- MSE (Mean Squared Error)
- RMSE (Root Mean Squared Error)
- MAE (Mean Absolute Error)
- R2 Score
- Cross-validation score

## Выбор модели

Система обучает три типа моделей одновременно:

1. Линейная регрессия - быстрая, интерпретируемая
2. Дерево решений - нелинейные зависимости, интерпретируемая
3. Нейронная сеть - сложные паттерны, высокая точность

После обучения автоматически выбирается лучшая модель по метрике R2 Score. Все три модели сохраняются, и можно посмотреть их метрики через команду `summary`.

## Примечания

- При первом запуске система обучит начальную модель на первом батче данных
- При последующих запусках система будет обрабатывать следующие батчи и обновлять модель
- Все версии моделей сохраняются и доступны для анализа
- Система автоматически выбирает лучшую модель из трех типов на основе метрик
- Для inference необходимо сначала обучить модель через режим `update`
- Входные данные для inference должны содержать те же признаки, что использовались при обучении
